# Міграція: матч-центр → новини

Інструкція для переходу зі старої схеми (окремі сторінки `/matches/[slug]` з Markdown і YouTube) на нову (запис матч-центру + повний контент у новині).

---

## Що змінилось

| Було | Стало |
|------|--------|
| Сторінка матчу `/matches/{slug}` | Клік «Матч-центр» → `/news/{postSlug}` |
| Колекція `matchPages`: команди, рахунок, `descriptionMarkdown`, YouTube | `matchPages`: лише `title`, `tour`, `postSlug`, `published`, `updatedAt` |
| YouTube на матч-сторінці / у «Попередньому матчі» | YouTube лише в новині (`youtubeVideoId`) |
| Поле `matchPageSlug` у featured-матчах | Поле `postSlug` (старе `matchPageSlug` ще **читається**, але при збереженні пишеться `postSlug`) |

Старі URL `/matches/...` більше не працюють (404). Контент потрібно перенести в новини.

---

## Передумови

1. Код з оновленням задеплоєно.
2. Є доступ до [Firebase Console](https://console.firebase.google.com) (Firestore) або до адмінки сайту.
3. У `.env.local` / на хостингу налаштовані `ADMIN_NEWS_SECRET`, Firebase Admin (для серверних сторінок).

---

## Крок 1. Новини (контент матчів)

Для кожного матчу, який раніше мав окрему сторінку:

1. Відкрийте **Адмін → Адмін новин** (`/admin/news`).
2. Створіть новину (або відкрийте вже існуючу, якщо текст там уже є).
3. Перенесіть у поля:
   - **Заголовок / опис / обкладинка** — як для звичайної новини.
   - **Текст у Markdown** — колишній `descriptionMarkdown` зі сторінки матчу (скопіюйте з Firestore або з чернетки в адмінці, якщо ще зберегли).
   - **Посилання YouTube (опц.)** — якщо на старій сторінці було відео; плеєр з’явиться в кінці статті на `/news/{slug}`.
4. Опублікуйте новину і **запишіть slug** (у URL: `/news/ulychne-vs-kolos-...` → slug = `ulychne-vs-kolos-...`).

Повторіть для всіх матчів, на які посилаються кнопки «Матч-центр».

---

## Крок 2. Записи матч-центру (`matchPages`)

### Варіант A — через адмінку (рекомендовано)

1. **Адмін → Матч-центр** (`/admin/match-pages`).
2. Старі записи **без `postSlug`** у Firestore **не показуються** на сайті (маппер їх ігнорує). У списку адмінки вони можуть бути відсутні — тоді створіть запис заново.
3. Для кожного матчу натисніть **Додати запис** або **Редагувати**:
   - **Назва матчу** — коротка назва для списку (наприклад, `«Уличне» — «Колос»`).
   - **Тур** — номер туру, якщо потрібно.
   - **Новина** — оберіть новину з кроку 1.
   - **Опубліковано** — увімкніть, коли новина готова.
4. Збережіть. Перевірте на `/matches` у блоці «Останні матчі»: клік має вести на `/news/{slug}`.

### Варіант B — напряму в Firestore

Колекція: **`matchPages`**

Приклад **нового** документа:

```json
{
  "title": "«Уличне» — «Колос»",
  "tour": 18,
  "postSlug": "ulychne-vs-kolos-match-report",
  "published": true,
  "updatedAt": "2026-05-27T12:00:00.000Z"
}
```

Обов’язкові поля для відображення на сайті: `title`, `postSlug`, `published: true`.  
Документ зі старими полями (`homeTeam`, `awayTeam`, `descriptionMarkdown`, `slug` тощо) **без `postSlug`** на публічному сайті не з’явиться.

Можна **не видаляти** старі поля одразу — вони просто ігноруються. Після перевірки документ можна спростити вручну або залишити як архів.

## Крок 3. Featured-матчі (наступний матч / календар)

Колекція: **`matches`** (зазвичай один документ, напр. `featured` — перевірте `MATCHES_DOC_ID` у `src/lib/matches.ts`).

1. Відкрийте **Адмін → Адмін матчів** (`/admin/matches`).
2. У блоці **Наступний матч** і кожному **майбутньому матчі**:
   - Поле **«Новина матч-центру»** — оберіть відповідну новину (замість старої «Сторінки матч-центру»).
   - Дата, команди, рахунок залишаються як раніше (для блоків на `/matches`).
3. Приберіть з Firestore поле **`youtubeVideoId`** у цих матчах, якщо воно ще є — відео тепер лише в новині.
4. Збережіть форму.

У Firestore після збереження з’явиться **`postSlug`**. Старе **`matchPageSlug`** код ще читає як запасний варіант, але нові зміни краще зберігати лише через адмінку.

---

## Крок 4. Новини — поле YouTube

Колекція: **`posts`**, документ = slug новини.

Додайте за потреби:

```json
{
  "youtubeVideoId": "dQw4w9WgXcQ"
}
```

(11-символьний ID, не повний URL — у адмінці новин URL парситься автоматично.)

---

## Чеклист після міграції

- [ ] Кожен матч зі списку «Останні матчі» відкриває правильну новину.
- [ ] Кнопка «Матч-центр» у «Наступний матч» веде на `/news/...`, не на `/matches/...`.
- [ ] У новинах з відео внизу є компактний плеєр.
- [ ] Старі посилання `/matches/old-slug` більше не використовуються в соцмережах / розсилках (або оновлені на `/news/...`).
- [ ] У `matchPages` немає опублікованих записів без існуючого `postSlug` у `posts`.

---

## Типові проблеми

**Запис матч-центру не видно на `/matches`**  
- Перевірте `published: true` і наявність `postSlug`, що відповідає існуючій новині в `posts`.

**Помилка при збереженні в адмінці матч-центру: новину не знайдено**  
- Спочатку створіть і опублікуйте новину; slug у Firestore (`posts/{slug}`) має збігатися з обраним у списку.

**Кнопка «Матч-центр» веде на `/matches` без переходу на новину**  
- У featured-матчі не задано `postSlug` — оновіть через `/admin/matches`.

**Контент старої сторінки матчу зник**  
- Він не мігрований автоматично. Скопіюйте `descriptionMarkdown` зі старого документа `matchPages` у тіло новини (крок 1).

---

## Порядок робіт (коротко)

1. Створити новини з Markdown (+ YouTube за потреби).  
2. Оновити / створити записи в `matchPages` з `postSlug`.  
3. Прив’язати featured-матчі до тих самих новин у `/admin/matches`.  
4. Перевірити сайт і оновити зовнішні посилання.

Орієнтовний час: **15–30 хвилин** на один матч (залежить від обсягу тексту і фото).
